WPtool Splunk Tool Summary

This file summarizes the current Splunk tab behavior so it can be reimplemented in another program.

1. Main UI layout
- The first tab is a Splunk search tool.
- Top-left area contains an Index selector.
- Top-right area contains:
  - Text (literal) input
  - Start time input
  - End time input
- Below the inputs:
  - Generated Query area
  - Splunk Results area
- Toolbar buttons:
  - Run Search
  - Splunk Setting
  - Index Setting

2. Index input behavior
- Index is no longer a plain text box.
- It is a dropdown/combobox.
- The dropdown values come from a local text file:
  - index_settings.txt
- Each non-empty line becomes one selectable index option.
- Example:
  - If index_settings.txt contains:
    A1
    B2
    C3
  - Then the Index dropdown shows:
    A1, B2, C3
- If the saved index list changes, the dropdown is refreshed immediately after saving.
- On next program launch, the saved list is loaded automatically.

3. Index Setting popup
- Button name: Index Setting
- Opens a separate popup window.
- Popup contains:
  - Title
  - Multi-line textarea
  - Save button
- Purpose:
  - User enters one index per line.
- Save behavior:
  - Content is saved locally to:
    index_settings.txt
  - The content is loaded automatically on next launch.
  - Saved lines are used to populate the Index dropdown.

4. Splunk Setting popup
- Button name: Splunk Setting
- Opens a separate popup window.
- Popup is larger than before and resizable.
- Popup contains:
  - SPLUNK_HOST
  - SPLUNK_USERNAME
  - SPLUNK_PASSWORD
  - Save button
- Save behavior:
  - Content is stored locally in:
    splunk_settings.json
  - Data is loaded automatically when the program starts.

5. Search text behavior
- Text field label is:
  Text (literal)
- The entered value is treated as a literal string search term.
- The value is escaped before being added to the query string.

6. Time input behavior
- Time inputs are:
  - Start
  - End
- User input format:
  YYYYMMDD HH24MISS
- Example:
  20260403 153000
- Validation rules:
  - Start must match the exact format.
  - End must match the exact format.
  - Start must be earlier than or equal to End.
- If validation fails:
  - Search does not run.
  - Error message is shown in the results area/status.

7. Generated Query behavior
- Generated Query is shown as a single line.
- Font size is smaller than before.
- It is only a preview string for the user.
- The preview query currently looks like:
  search index="A1" "error OR failed" earliest="4/3/2026:15:30:00" latest="4/3/2026:16:00:00" | table _time host source sourcetype _raw

8. Actual Splunk login/search flow
- Search now uses a two-step session-key flow.

Step 1. Login
- Login URL:
  http://smiot.samsungds.net:8089/services/auth/login
- Username/password are posted to that endpoint.
- Response is XML.
- sessionKey is extracted from the XML response.

Step 2. Search
- Search endpoint is built from the login host:
  http://smiot.samsungds.net:8089/services/search/jobs/export
- Authorization header format:
  Authorization: Splunk <sessionKey>
- Search request is sent with:
  - search
  - earliest_time
  - latest_time
  - output_mode=json
  - exec_mode=oneshot

9. Time sent to Splunk
- User enters:
  YYYYMMDD HH24MISS
- Internally converted to datetime.
- For preview query:
  converted to Splunk-style absolute time string, e.g.
  4/3/2026:15:30:00
- For actual REST search:
  converted to epoch seconds and sent as:
  - earliest_time
  - latest_time

10. SSL behavior
- Default:
  SPLUNK_VERIFY_SSL=true
- If SPLUNK_VERIFY_SSL is not one of:
  1, true, yes
  then SSL certificate verification is disabled for the request.
- So current behavior is secure by default.

11. Search result behavior
- Splunk Results area is a multi-line text output box.
- Raw JSON line output from Splunk export is parsed line by line.
- Up to 20 result lines are formatted and shown.
- If more than 20 results are returned:
  a final line indicates that more events exist.
- If no results:
  "No results returned from Splunk."

12. Local files currently used
- splunk_settings.json
  - Saves Splunk connection settings
- index_settings.txt
  - Saves selectable index list
- splunk_tab_injection.txt
  - Full code-style copy/paste source for the Splunk tab

13. Current important implementation notes
- Index dropdown is populated from index_settings.txt, not typed freely.
- Splunk search uses sessionKey auth, not only Basic auth.
- The login URL is fixed to:
  http://smiot.samsungds.net:8089/services/auth/login
- Generated Query is preview-only.
- Search execution uses validated times and REST parameters.

14. If reimplementing in another program
- Recreate these 3 storage concepts:
  - connection settings JSON
  - index list text file
  - UI state in memory
- Recreate these 2 popups:
  - Splunk Setting
  - Index Setting
- Recreate these 2 flows:
  - load saved settings on startup
  - sessionKey login first, search second
